<h1 align="center"> Routify LLM </h1>

##  简介
Routify-LLM 是专为 Java 生态打造的 LLM 模型网关 SDK。

引入 Routify-LLM，它将会给你的模型调用层带来生产级的稳定性、经济性与可观测性，让每一次模型调用都稳如磐石、省到极致，且对业务代码完全透明。

它原生支持以 OpenAI、Claude、Google 等主流协议传入请求，内置协议转换引擎自动适配通义千问、文心一言、Gemini 等厂商接口；同时基于实时延迟、错误率、成本等指标动态路由，并集成熔断、限流、重试、故障转移等高可用机制。

> 说人话版：一套代码调不同模型 + 帮你将生产环境该有的那些破事全包了，让你专注于业务逻辑

## 它能做什么

### **直接作为模型调用层（新项目）**
觉得 LangChain4j 或其他框架没有完善的路由、重试机制？直接用 Routify-LLM 的 SDK 完成所有模型调用。它提供完整的 Chat/Completion 接口，一套代码通吃全网模型，同时内置路由、容灾、配额、计量等治理能力，让你的项目从第一天起就拥有生产级模型调用层。

### **给现有框架加上高可用（老项目）**  
正在用 LangChain4j、Spring AI 但担心厂商挂了服务会断？Routify-LLM 可以作为底层模型源，透明接入你的代码，自动处理重试、熔断、故障转移，让原有项目瞬间拥有生产级保障，一行调用代码都不用改。

### **快速搭建多协议接入网关**
不用自己写 OpenAI 转通义、Claude 转文心的适配代码。Routify-LLM 直接接收主流协议请求，自动转换后调用目标模型，你只管配好厂商信息，剩下的它搞定。

### **智能路由与高可用**  
实时监控延迟、错误率、成本，动态选择最优模型接口；内置熔断、限流、重试、故障转移机制，厂商抖动或配额耗尽时毫秒级切换，业务无感知。

### **多 Key 配额与成本管控**  
给不同团队/业务线分配独立虚拟 Key，设置月度预算、绑定可用模型；实时统计 token 消耗和费用，月底一键导出报表，成本透明可控。

### **全链路可观测与问题排查**  
  每笔请求从入口到厂商响应的完整链路都有日志，哪个环节慢了、哪个厂商报错了，一眼定位。再也不用靠猜 debug。
